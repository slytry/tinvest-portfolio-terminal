## tinvest-portfolio-terminal

Инструмент для анализа портфеля в Тинькофф Инвестициях. Проект забирает данные по позициям из API Тинькофф, обогащает их информацией по облигациям (доходность к погашению, рейтинг, дюрация и т.п.) и показывает агрегированную картинку по портфелю:

- распределение по классам активов (акции / облигации / фонды / валюта);
- стоимость портфеля и доля каждой позиции;
- расширенные данные по облигациям (YTM, рейтинг, срок до события/погашения, купон, цена, объем выпуска);
- переключение между разными счетами (если у пользователя несколько аккаунтов).

Стек: React, React Router (Vite под капотом), Mantine UI, TypeScript.

---

## Требования

- Node.js **25.8.0** (как указано в `package.json`)
- npm (идет вместе с Node)

Рекомендуется использовать [`nvm`](https://github.com/nvm-sh/nvm) для установки нужной версии Node.

```bash
nvm install 25.8.0
nvm use 25.8.0
```

## Установка зависимостей

Клонируем репозиторий и устанавливаем зависимости:

```bash
git clone https://github.com/slytry/tinvest-portfolio-terminal.git
cd tinvest-portfolio-terminal

npm install
```

## Запуск приложения в режиме разработки

```bash
npm run dev
```

Эта команда запускает dev-сервер React Router (на базе Vite). В терминале будет указан адрес, обычно это что-то вроде:

- http://localhost:5173

Открой этот адрес в браузере.

> Важно: запускать именно через `npm run dev`, а не напрямую `vite` или `node vite.config.ts`.

---

## Сборка и запуск продакшн-версии

1. Собрать проект:

```bash
npm run build
```

2. Запустить собранный сервер:

```bash
npm start
```

Команда `start` использует `react-router-serve ./build/server/index.js` и поднимает сервер для продакшн-сборки.

## Обновление данных по облигациям

В папке `scripts/bonds-dohod` лежат скрипты для загрузки и конвертации данных по облигациям из внешнего источника (dohod.ru). Для обновления данных используется один агрегирующий скрипт:

```bash
npm run dohod:update
```

Он последовательно выполняет:

- `npm run dohod:load` — загрузка сырых данных;
- `npm run dohod:convert` — конвертация в пригодный для фронтенда формат.

Результат обычно кладется в `public/data` (например, `bonds_latest.json`) и далее подхватывается приложением.

## Типичные проблемы и их решение

### 1. Ошибка `ERR_REQUIRE_ESM` при старте

Сообщение вида:

```text
Error [ERR_REQUIRE_ESM]: require() of ES Module .../node_modules/vite/dist/node/index.js from .../vite.config.ts not supported
```

Причины и решения:

- Запускать проект нужно **через npm-скрипт**:

  ```bash
  npm run dev
  ```

  а не напрямую `vite` или запуском файла `vite.config.ts` через Node.

- Убедись, что версия Node соответствует требуемой (`25.8.0`) и не используется сильно устаревшая глобальная версия.

### 2. Ошибка Mantine `Duplicate options are not supported`

Сообщение вида:

```text
[@mantine/core] Duplicate options are not supported. Option with value "Unknown" was provided more than once
```

Эта ошибка возникала из-за того, что несколько аккаунтов имели одинаковое имя (`Unknown`), и селект аккаунтов использовал это имя как `value`. В коде это исправлено: теперь селект использует уникальные `id` для `value`, так что дубликаты не появляются.

Если ты форкаешь/меняешь проект и снова столкнулся с подобным — проверь, что `value` у всех опций Mantine Select уникально.

## Скрипты из `package.json`

Основные команды:

- `npm run dev` — запуск dev-сервера;
- `npm run build` — сборка приложения;
- `npm start` — запуск продакшн-сборки;
- `npm run typecheck` — генерация типов роутера и проверка TypeScript;
- `npm run check` — линт/проверка кодовой базы (ultracite/biome);
- `npm run fix` — автоисправление форматирования/замечаний линтера;
- `npm run dohod:load` — загрузка данных по облигациям;
- `npm run dohod:convert` — конвертация загруженных данных;
- `npm run dohod:update` — выполнить загрузку + конвертацию подряд.

---

## Структура проекта (кратко)

- `app/` — основное приложение React Router
  - `routes/` — маршруты (страницы)
  - `pages/portfolio/` — отображение портфеля, таблицы, визуализация долей
  - `components/` — переиспользуемые UI-компоненты
  - `api/` — клиентские обертки над API, состояние (zustand) и типы
- `public/` — статика и подготовленные JSON-данные
- `scripts/bonds-dohod/` — вспомогательные скрипты для работы с данными по облигациям

---

## Как начать работу над проектом

1. Установить Node 25.8.0 (через `nvm` или иным способом).
2. Клонировать репозиторий и выполнить `npm install`.
3. Запустить приложение командой `npm run dev`.
4. Открыть в браузере адрес из терминала (обычно `http://localhost:5173`).
5. При необходимости обновить данные по облигациям через `npm run dohod:update`.

После этого приложение готово к локальной разработке и экспериментам с портфелем.
